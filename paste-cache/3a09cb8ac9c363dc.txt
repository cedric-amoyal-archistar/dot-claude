In UploadCustom.vue we currently add the file data to an array using addFile(), then later on in the process we trigger createSubmission().
In createSubmission() we first upload all the files to S3 and when we are done we are actually creating the submission with "await this.$APIHelper.EcheckAPI.post('/designs', payload);" the payload includes the awsKey of the files uploaded earlier. This is working well.
Now we want to add a virus check.
The new flow will be to upload the files to S3 straight away and then to trigger the virus check.
The logic to upload to S3 is in the await Promise.all() in createSubmission (but we will now only trigger it one file at the time)
The virus check is done using
POST /[designs/design-checks]/design-upload-url
It is not instant so we need to poll GET /[designs/design-checks]/design-upload-status?awsKey={aws_key} until we get the response.
If the file is fine then we want to add the s3_key to the awsKey property the same way we are currently doing it here this.selectedFiles[index]['awsKey'] = s3_key;
if there is an issue with the file we want to remove the file from selectedFiles and to display an error message toast.
Then the "await Promise.all(...)" in createSubmission() will not be needed but the rest of the createSubmission() logic should be the same.
